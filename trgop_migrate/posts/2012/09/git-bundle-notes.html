Git Bundle notes.

September 19 2012, 10:04 PM  by Victor Goff

This week I needed to send a git repository over to another machine without access to my repository.  I wanted to keep it intact.

So here comes the git bundle command to the rescue.

First I had to find the references that I needed to bundle, and I found that the hash for the initial commit was f787346 (short form) and then performed this command:

    git bundle create my_cool_project.bundle master f787e46

I then moved it to my desktop just to work on it for a moment and confirm that it was intact.

    mv my_cool_project.bundle ~/Desktop/
    cd ~/Desktop/

Once I had it in place, I decided to clone it (though I could have simply done a git ls-remote my_cool_project.bundle to check the references, or a git verify my_cool_project.bundle) right there on my desktop.

    git clone my_cool_project.bundle 

This will give an error if the branch doesn't exist, as I found out.  

    git clone my_cool_project.bundle -b master

This provided a master branch for me, and I am ready to go.

Remember, a .bundle file is simply a git repository that is 'bundled' so you are free to fetch or pull or merge or clone it.  You are also not restricted to naming it with a .bundle extension.  But it may help to idenitfy the file later.
