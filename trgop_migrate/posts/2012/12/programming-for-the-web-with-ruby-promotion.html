December 31 2012, 10:40 PM  by Victor Goff

We are gearing up for another fun and exciting 2 weeks of Programming for the Web With Ruby and I wanted to post it here, and let you guys know.

When letting folks know about the course, you can tweet the following, or post it on facebook, or in Google Plus or where ever it is that you hang out.

  • A 'Free' online course: #Programming the web with #Ruby. Registrations open ow.ly/eUdcZ

Satish has decided that he wants to give out my popular blog post about RubyDocTest.  And I thought it was a good idea.  I like simple tools especially for those that are learning, it reduces the yak (*) hair that gets on your tongue.  RubyDoctest is the "simple to use, stays out of your way, everyone uses IRB anyway, so you might as well use it for documentation and usage examples that prove themselves" testing framework with only 4 things to know about it.

Of course, you can read it here or download it or pass it on.

RubyDoctest.pdf View this file

 

 

(*) Shaving yaks, you can find out what this is, but the point here is that it is better to test immediately, well enough, rather than putting it off.  Later, you can learn testing frameworks and you might even still finding yourself using rubydoctest for quick explorations, and maybe even leaving them to document those explorations, even if you are using minitest or rspec or cucumber or any other number of more robust testing tools.
